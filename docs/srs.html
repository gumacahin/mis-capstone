Introduction

The project aims to develop a web-based application called “WSIDT” that serves as a task-management and productivity-tracking tool for the University of the Philippines Open University (UPOU) community. The objective is to evaluate its usefulness for increasing productivity among UPOU faculty and staff.

For more information, see the project proposal.

Use Cases

The software application has two actors: users and administrators. User are the primary actors. Users are able to register and sign in to the application. Once signed in users are able to create todos and organize them. Administrators are users given administrative roles and are able to login to the administrative section of the application and able to manage various aspects of the app.

Usability Requirements
User Interface Design

Consistency is a paramount requirment in user interface design. Throughout the application, the design should maintain a cohesive and unified appearance. This consistency extends to color schemes, fonts, and layout elements.

The user interface must be intuitive. Users should be able to perform common tasks seamlessly without the need for extensive training. Navigation elements and interactions should be clear and self-explanatory.

The user interface should be responsive. The application should promptly respond to user actions, offering immediate feedback.

Navigation

Efficient task entry is a core aspect of the system's usability. Adding a new task should be a straightforward process, achievable with a maximum of two clicks. The task entry form must be easily accessible from any screen within the application.

Managing tasks should be intuitive. Users should be able to effortlessly edit or delete tasks through a clear and accessible interface. Support for drag-and-drop functionality for task prioritization and sorting is essential.

Accessibility

The web-application must be compatible with major web browsers, including Chrome, Firefox, Safari, and Edge. The system should be mobile responsives.

Accessibility considerations should be integrated into the design. In particular, customizable font size settings should be available to the user.

Reliability and Uptime

The web-app's server should have an uptime of 99.9%. The web-application itself should be compatible with modern browsers running on relatively recent desktop, laptops and mobile devices.

Security Requirements

The security of the web-app will be checked against the OWASP Top Ten.

Hardware Requirements

A majority of the users will be using their work desktop or laptop. Modern work machines have enough computing power to run day-to-day tasks including modern browers from which the web-application will be accessed. Outside of their work machines, users will access the application through a mobile browser on their personal smart phones.

System Software Requirements?

The web-application require an modern browser running on desktop, laptop or mobile device.
