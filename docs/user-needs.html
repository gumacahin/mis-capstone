Glossary
Process impact: The statement of user needs documents and explains the actual desires of stakeholders in roughly their own words. What they desire is never exactly what the product provides. Documenting user needs here, independently from the SRS, helps to keep the SRS precise and makes the tasks of verification and validation more effective. This document is not an informal draft of the SRS, it is different document with a complementary purpose.

What is the system's business environment?
The UPOU faculty needs to manage their time between research, extention and administrative work. A productivity tracker could help keep track of these activities and help identify which category of activities they should focus on.
UPOU administrators needs to measure and keep track of faculty productivty in order to identify areas of improvement.
What is the system's physical environment?
This system will not run on a physical web server but will instead be hosted one the cloud. In particular, the applicaton will be deployed in Laravel Forge. Users of the system will be using desktop or mobile browsers access it.
What is the system's technology environment (hardware and software)?
A majority of the users will be using their work desktop or laptop. Modern work machines have enough computing power to run day-to-day tasks including modern browers from which the web-application will be accessed. Outside of their work machines, users will access the application through a mobile browser on their personal smart phones.

All
All stakeholders share the following key needs:
  1. Synced workflow across their devices
  2. Confidence over the safety and security of their personal information
User
User's want to see the value of using the app. That means the app should be able to fulfill their needs and be familar and intuitive enough so as not to have a learning curve that could make them shy away from the app.

Key needs:

  1. Easily figure out how to use the key features of the app
  2. Understand how they are utilizing their time and use it to organize tasks
  3. Easily access documentation should they have any questions
Administrators
Administrators want to be able to see the productivity of the faculty and staff. They want to be able to see the data in a way that would allow them to make decisions on how to improve productivity.

Key needs:

  1. View metrics that would give insights to future plans and events
  2. View time-keeping data that would allow them to better manage schedules
  3. Understand overall productivity of the workforce and ways to improve
