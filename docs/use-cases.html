UC-00: User registration

Summary: Users need to register to the System in order to use it.
Priority: Essential
Use Frequency: Once per user
Prereq:
User is not logged in.
Main Success Scenario:
  1. visit Registration page
  2. enter identifying information: name, email, password, password, password confirmation
  3. submit form
  4. see Dashboard page
Alternative
Scenario Extensions:
  • If any of the fields failed validation or the email is taken, an error will be displayed and the registration will not proceed.
Notes and Questions
  • Subsequent releases should in implement email verification.

UC-01: User login

Summary: User needs to be logged in to use the System.
Priority: Essential
Use Frequency: Always

User is not logged in.
Main Success Scenario:
  1. visit Sign In page
  2. fill out username and password

UC-02: Reset password

Summary: If a user forgets their password, they can for a password reset link to be emailed to them.
Priority: Essential
Use Frequency: Rarely
Prereq:
User is not logged in.
Main Success Scenario:
  1. visit Forgot Password page
  2. enter email
  3. submit form
  4. see "password reset email sent" message
  5. receive password reset email
  6. click "password reset"
  7. fill out password and password confirmation fields
  8. submit form
  9. see login page
  10. perform user login using new password
Alternative
Scenario Extensions:
  • If email is not registered in the System, form will display an error.
  • If user requested a password request recently, they will be asked to wait before trying again.

UC-20: Add task

Summary: Users can add tasks.
Priority: Essential
Use Frequency: Often
Main Success Scenario:
  1. open add task dialog from toolbar menu
  2. enter task information: task name, task description
  3. optionally enter additional information: date, time, priority, labels
  4. submit form
  5. see "task added notification" message
Alternative
Scenario Extensions:
  • If any of the form fields failed validation, an error be displayed and the task will not be saved.
  • Aside from the add task button on the toolbar, the add task form can also be opened inline in various sections of the System.
Notes and Questions

    The System displays task lists various sections. The add task option is generally available whenever such list displayed. See UC-31, UC-32, UC-40, UC-50, UC-60, UC-70

Notes and Questions

UC-22: Edit task

Summary: Users can edit individual tasks.
Priority: Essential
Use Frequency: Often
Main Success Scenario:
  1. click edit task on a task to open inline form
  2. update one or more fields: name, description, due date, priority, labels
  3. submit form
  4. see task updated notification
Alternative
Scenario Extensions:
  • Editing can also be initiated by clicking on the task name or task description after opening a task for viewing (UC-21).
  • Editing can be aborted and changes to the task discarded by clicking on the cancel button on the editing form.
  • If any of the form fields failed validation, an error will be displayed and the task will not be saved.
